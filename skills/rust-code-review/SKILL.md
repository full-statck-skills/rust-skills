# Rust 代码审查

> Rust 代码审查指南，基于官方编码规范与社区 11,500+ 源码文件的模式分析。

## Capability Boundaries

### ✅ 强项
1. Rust 代码风格与规范（Rustfmt、Clippy 规则）
2. 常见反模式与错误
3. unsafe 代码安全审查
4. 性能热点识别
5. 并发安全问题（数据竞争、死锁）
6. API 设计审查
7. 错误处理合理性
8. 所有权/生命周期使用正确性

### ⚠️ 前置要求
1. 代码可通过 `cargo check` 编译

### ❌ 不适用范围
1. 语法教学 → 使用 `rust-core` 技能

## 审查清单

### 安全性
- [ ] unsafe 代码是否有 safety 文档
- [ ] unsafe 块是否尽可能小
- [ ] 裸指针操作是否满足安全不变条件
- [ ] FFI 调用是否正确处理错误码

### 正确性
- [ ] 错误是否被正确处理（不是都 unwrap）
- [ ] 是否使用了 ? 运算符传播错误
- [ ] 匹配是否穷尽
- [ ] 泛型约束是否合理

### 性能
- [ ] 热点路径是否避免不必要的分配/克隆
- [ ] 迭代器是否使用了惰性方法
- [ ] 大型数据类型是否使用了 Box/Arc 而非栈复制

### 并发
- [ ] Send + Sync 实现是否正确
- [ ] 锁范围是否最小化
- [ ] 是否避免了死锁

## Data Privacy

本技能不收集、存储或传输任何用户数据。
